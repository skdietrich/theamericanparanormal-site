How It Works

The Signal Pipeline

Six stages from input to event record. Each stage is mathematically defined and auditable.

1

Input Stream

Microphone captures real acoustic air-pressure fluctuations, or a synthetic noise source for pipeline testing. Sampled at 44.1kHz, processed in 500ms windows.

2

Baseline Calibration

Instrument collects 120 windows (60 seconds) of signal data. Computes mean and variance — or full 16×16 covariance matrix — of the undisturbed environment. Frozen. Never modified during a live session.

3

Feature Extraction

Each 500ms window produces a 16-dimensional feature vector: entropy, variance, drift, runs, autocorrelation, jitter, spectral flux, RMS amplitude, DC bias, tail weight, burst density, silence fraction, phase coherence, roughness, kurtosis, and spectral novelty.

4

Anomaly Scoring

Mahalanobis distance measures how far each window is from the calibrated baseline.

D² = (x − μ)ᵀ Σ⁻¹ (x − μ)
score = √(D²/k), k = 16

Under the null hypothesis, D² follows χ²₁₆. The p-value is the chi-squared survival probability.

5

CUSUM Persistence

A single anomalous window proves nothing. The CUSUM accumulator rewards sustained deviation across multiple windows while decaying in the absence of new evidence.

Cₜ = max(0, Cₜ₋₁ + eₜ − 0.72)

Isolated spikes cannot qualify. Consistent, repeated elevation is required.

6

Event Logging

A qualified event requires: score ≥ threshold for 3 consecutive windows, CUSUM ≥ 3.0, 15+ seconds since the prior event, and no operator freeze active. Each record includes all 16 feature values, exact p-value, CUSUM, timestamp, and session type.

The 16 Signal Features

What the Instrument Measures

entropy

Shannon Entropy

Frequency distribution randomness. Persistent low entropy in broadband noise is anomalous.

drift

Mean Drift

Signal stationarity within the window. Sustained drift indicates non-stationarity.

variance

Amplitude Variance

Spread of signal amplitude values within the 500ms window.

runs

Zero-Crossing Rate

Fraction of sign flips. Low runs = smooth DC-like signal, anomalous in noise.

autocorr

Autocorrelation

Lag-1 self-similarity. High autocorrelation suggests persistent periodic structure.

jitter

Timing Jitter

Irregularity of oscillation rhythm across sub-blocks within the window.

flux

Spectral Flux

Rate of change of frequency content between consecutive windows.

kurtosis

Kurtosis

Tail weight of amplitude distribution. Gaussian noise ≈ 0; impulsive noise is high.

novelty

Spectral Novelty

Distance of current spectrum from rolling mean of recent windows.

phase

Phase Coherence

Regularity of zero-crossing intervals. Low = regular oscillation, high = chaotic phase.

tail

Tail Fraction

Samples exceeding 3σ. Measures heavy-tailed statistical outliers within the window.

burst

Burst Density

Fraction of samples near peak amplitude. Episodic high-amplitude events.

The statistics describe deviation. They do not by themselves identify the cause. The instrument marks candidate moments for review — so patterns can be studied with discipline and humility.

Field Protocol

The Correct Workflow for a Session

Every live investigation should be paired with a matched control session. That comparison is where meaningful data lives.

Before You Arrive

  • ✓ Charge laptop fully
  • ✓ Test on synthetic source at home
  • ✓ Plan one control session and one live session at the location
  • ✓ Document environmental notes: HVAC, traffic, equipment in use

Control Session (Run First)

  • ✓ Declare session type: CONTROL
  • ✓ Select signal source: Audio (Mic)
  • ✓ Calibrate 60 seconds. Do not speak or move.
  • ✓ Start Live Watch. Leave the room or remain completely still for 20+ min.
  • ✓ Stop and export. Note any anomalies.

Live Session (Run Second)

  • ✓ Declare session type: LIVE
  • ✓ Re-calibrate in the same position
  • ✓ Start Live Watch
  • ✓ Conduct your investigation. Use Mark Moment for key activities.
  • ✓ Remain away from laptop — every touch triggers the contamination guard
  • ✓ Stop and export. Note timestamps of all activities.

After the Investigation

  • ✓ Compare live and control session statistics externally
  • ✓ Verify qualified event p-values are lower in the live session than control
  • ✓ Check the methods document for any drift warnings
  • ✓ Archive all four export files: JSON, CSV, HTML, Methods
  • ✓ The methods document is designed for independent technical review

The key principle: A single live session, however dramatic, proves nothing in isolation. What matters is whether the distribution of anomaly scores, event rates, and p-value distributions differs between live and control sessions conducted at the same location with the same equipment over equal durations. That comparison is what the session type lock is designed to protect.
